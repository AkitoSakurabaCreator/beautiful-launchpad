# Beautiful Launchpad（macOS ネイティブの美しいランチャー）

[English](README.en.md) | **日本語**

macOS 26 (Tahoe) で廃止された従来の **Launchpad** を、ただ復活させるだけではありません。
**従来どおりのクラシックな見た目**にも、**背景・テーマ・レイアウトを自分好みに仕立てたカスタムデザイン**にもできる、
**美しい**ネイティブ macOS ランチャーです。インストール済みアプリを全画面グリッドに並べ、検索・ページ送り・
フォルダ・ドラッグ並べ替えに対応し、クリックで**実際にアプリを起動**します。

- 言語/技術: **Swift 5.9+ / SwiftUI + AppKit**（SwiftPM。フル Xcode 不要、Command Line Tools のみでビルド可）
- 動作確認環境: macOS 26.5.1 / Apple Silicon (arm64)

---

## 特長（クラシック再現 ＋ 自由なカスタマイズ）

- 🖥 **全画面フロストガラス背景**（`NSVisualEffectView` の behind-window ブラーでデスクトップをぼかす）
- 🔍 **インスタント検索** — 起動直後から入力するだけで全アプリを横断フィルタ
- 🗂 **実アプリの自動走査** — `/Applications`・`/System/Applications`・`~/Applications` 等を走査し、
  アイコン付きで一覧（サブフォルダの同梱アプリも 1 階層まで拾う）
- 📄 **複数ページ + ページドット** — スクロール／←→キー／ドットクリックで移動
- 📁 **フォルダ** — アプリを別アプリの**中央**へドラッグで作成、フォルダへドラッグで追加、
  オーバーレイで開いて起動・名称変更・取り出し
- 📂 **入れ子フォルダ（サブフォルダ）** — フォルダを開いた中で項目を別項目の**中央**へドラッグすると
  サブフォルダを作成。タップで階層を掘り下げ、**‹ 戻る**ボタンで一段戻る
- 📑 **フォルダのページ送り** — フォルダ内の項目が増えると自動で複数ページに分割
  （ページドット＋左右端へのドラッグで移動）。カードは項目数に関係なく一定サイズ
- ✋ **ドラッグ並べ替え** — アイコンの**端**付近へドロップで並べ替え、ページ端でページ送り
- 💾 **配置の永続化** — 並び順・フォルダを `~/Library/Application Support/Launchpad/layout.json` に保存
- 🚀 **クリックで実起動** → 起動後 Launchpad は自動で閉じる
- 🎞 **開閉アニメーション** — フェード＋ズームで滑らかに開閉（閉じる時もパッと消えない）
- 🖥 **マルチモニター対応** — 起動時に**カーソルのある画面**で開く
- 🎨 **外観カスタマイズ** — 背景（デスクトップぼかし／テーマ／画像／単色）、列数・行数、暗さ、アイコン名表示の切替
- 🧩 **ユーザー定義の項目** — **アプリ／スクリプト・コマンド／URL** を手動で追加（専用アイコンも設定可）。背景やアイコンを右クリック →「項目を追加」から登録
- 🖱 **右クリックメニュー** — 追加項目の編集／削除、アプリを **Finder で表示**、項目の**非表示**、その場で項目を追加
- 🙈 **項目の非表示 / 再表示** — 右クリック →「**非表示にする**」でアプリやカスタム項目をグリッド・検索から隠す
  （再スキャンしても復活しません）。**設定の「非表示の項目」**から「表示」／「すべて表示」でいつでも元に戻せる
- 🏷 **フォルダの色分け** — フォルダにプリセットカラー（9色＋デフォルト）を設定
- 🌐 **多言語対応** — 日本語／English／システム追従を **再起動なしで即時切替**
- ⬆️ **自動アップデート** — 起動時に自動でチェック（Sparkle）。設定パネルから手動チェックも可
- 📦 **設定の export / import** — 並び順・フォルダ・外観を JSON に書き出し、**別 Mac で読み込み**
- ⌨️ **Esc** で（設定/フォルダを閉じる → 検索クリア →）終了、空白クリックで終了

## ダウンロード（ビルド済みアプリ）

最新のビルド済み `Beautiful Launchpad.app` は [Releases](../../releases) から入手できます
（`v*` タグの push 時に GitHub Actions が自動ビルドして添付します）。

1. Releases から最新リリースの `.zip` をダウンロードして解凍（資産名はリリースにより異なります）
2. `Beautiful Launchpad.app` を `/Applications` などへ移動
3. **初回のみ**: ローカルビルドの **ad-hoc 署名**かつ未公証のため、ダウンロード版は
   Gatekeeper にブロックされます。次のいずれかで開いてください。
   - `Beautiful Launchpad.app` を **右クリック →「開く」** →「開く」を選択（初回のみ）
   - もしくはターミナルで隔離属性を外す:
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Beautiful Launchpad.app"
     ```

> 自分でビルドした `.app`（下記）はネット経由でないため、この操作は不要です。
> App Store 配布や Apple の公証（notarization）は行っていません（個人利用前提のため）。

## ビルド方法

```bash
# このディレクトリで
./build-app.sh            # release ビルド → Beautiful Launchpad.app を生成し ad-hoc 署名
# もしくは開発用に
./build-app.sh debug
```

生成物: `./Beautiful Launchpad.app`

### 起動

```bash
open "./Beautiful Launchpad.app"
# または Finder でダブルクリック
```

> 初回は Gatekeeper が「開発元を確認できません」と出る場合があります。その場合は
> **右クリック →「開く」**、または「システム設定 → プライバシーとセキュリティ」から許可してください
> （ad-hoc 署名のローカルビルドのため）。

### 開発時の素早い実行（バンドル無し）

```bash
swift run          # デバッグ実行
swift build        # コンパイルのみ
```

## 操作

| 操作 | 動作 |
|---|---|
| アプリのアイコンをクリック | そのアプリを起動して Launchpad を閉じる |
| 文字入力 | 検索（起動直後は検索欄に自動フォーカス） |
| ← / → キー、横スクロール、ドット | ページ移動 |
| アイコンをドラッグ（端へ） | 並べ替え。画面端でページ送り |
| アイコンを別アイコンの中央へドラッグ | フォルダ作成／フォルダへ追加 |
| フォルダをクリック | フォルダを開く（名称変更可） |
| フォルダ内で項目を**別項目の中央へドラッグ** | サブフォルダを作成／サブフォルダへ入れる（入れ子） |
| フォルダ内で項目を**中央以外へドラッグ** | フォルダ内で並べ替え |
| フォルダ内で項目を**左右端へドラッグ** | フォルダのページを送る |
| フォルダ内の項目を**パネル外（暗い背景）へドラッグ** | フォルダから出して一つ上の階層へ戻す（「−」ボタンでも可） |
| フォルダ内のサブフォルダをタップ／**‹ 戻る** | 階層を掘り下げる／一段戻る |
| アイコン／フォルダ／背景を**右クリック** | コンテキストメニュー（項目の追加・編集・削除、Finder で表示、**項目の非表示** など） |
| 右上の ⚙️ ボタン / **⌘,** | カスタマイズ画面を開く |
| Esc | 設定/フォルダを閉じる → 検索クリア → 終了 |
| 空白部分をクリック | 終了 |

配置をリセットしたい場合は、カスタマイズ画面の「リセット」、または
`~/Library/Application Support/Launchpad/layout.json` を削除してください。

## カスタマイズ / 設定の移行

右上の ⚙️（または **⌘,**）でカスタマイズ画面を開けます。

- **背景**: デスクトップぼかし（クラシック）／テーマ（6種のグラデーション）／画像（任意の壁紙を選択）／単色
- **暗さ**: 背景の上にかける暗幕の濃さ
- **レイアウト**: 列数（4〜10）・行数（3〜8）
- **アイコン名表示**: ラベルの ON/OFF
- **言語**: 日本語／English／システム追従（再起動なしで切替）
- **アップデート**: 起動時の自動チェック ON/OFF と「アップデートを確認…」での手動チェック

### 項目の追加 / フォルダの色

- **項目を追加**: 背景やアイコンを**右クリック →「項目を追加…」** で、アプリ／スクリプト・コマンド／URL を登録できます（アイコンも指定可）。スクリプト・コマンドはシェル経由で実行されます（要実行権限）。
- **フォルダの色**: フォルダを開いた画面のカラースウォッチから選択。先頭の「−」スウォッチでデフォルト（無色）に戻せます。

### 項目の非表示 / 再表示

- **非表示にする**: アプリやカスタム項目を**右クリック →「非表示にする」**で、グリッド・フォルダ・検索から隠せます。非表示にした項目は再スキャンしても再追加されません（フォルダ自体は非表示にできません）。
- **元に戻す**: 非表示の項目があるときだけ、カスタマイズ画面に**「非表示の項目」**セクションが表示されます。各項目の **「表示」**、またはまとめて **「すべて表示」** で元に戻せます。
- 非表示の選択は `layout.json` に保存されるため、アプリを再起動しても保持されます。

### 別の Mac へ移行（export / import）

- カスタマイズ画面の **「エクスポート…」** で、`launchpad-config.json`（並び順・フォルダ・外観）を書き出し。
- 別の Mac で **「インポート…」** から読み込むと復元できます。

> 注意：
> - レイアウトはアプリを**ファイルパス**で参照します。`/Applications`・`/System/Applications` の標準アプリは
>   どの Mac でも同じ場所のため移行できますが、**インポート先に存在しないアプリはスキップ**されます。
> - **壁紙画像はファイルパス参照**のため、画像自体は移行先にコピーされません（画像を同じ場所に置くか、移行先で選び直してください）。
> - 外観設定（テーマ・暗さ・列数等）は完全に移行されます。

設定ファイルの保存先:
`~/Library/Application Support/Launchpad/`（`layout.json` と `settings.json`）

## 構成

```
LaunchPad/
├── Package.swift                 # SwiftPM (executable target, macOS 13+)
├── Info.plist                    # .app バンドル用
├── build-app.sh                  # ビルド + .app 化 + ad-hoc 署名
└── Sources/Launchpad/
    ├── LaunchpadApp.swift        # @main / AppDelegate / 全画面ウィンドウ / キー・スクロール監視
    ├── Support.swift             # GridGeometry / VisualEffectView / chunk
    ├── Models.swift              # AppInfo / Folder / CustomItem / 永続化モデル（order・folders・customItems・hidden）
    ├── AppScanner.swift          # インストール済みアプリ走査 + アイコン取得
    ├── LaunchpadStore.swift      # 状態管理・ページング・検索・D&D・フォルダ（入れ子）・非表示・永続化
    ├── LaunchpadView.swift       # ルート / ページャ / グリッド / 検索 / ドット / 右クリックメニュー
    ├── IconViews.swift           # アプリ／フォルダのタイル描画（フォルダのミニプレビュー）
    ├── FolderViews.swift         # フォルダオーバーレイ（入れ子・ページ送り・色）
    ├── DragDrop.swift            # DropDelegate（中心=フォルダ / 端=並べ替え・ページ送り 判定）
    ├── CustomItems.swift         # ユーザー項目（アプリ/スクリプト/URL）の追加・編集オーバーレイ
    ├── SettingsView.swift        # カスタマイズ画面（背景・レイアウト・言語・更新・非表示項目・export/import）
    ├── Theming.swift             # 背景レンダリング（デスクトップぼかし／テーマ／画像／単色）
    ├── Localization.swift        # 多言語の in-code テーブル（日本語／English／システム追従・即時切替）
    └── Updater.swift             # Sparkle 自動アップデート（全画面オーバーレイ向けの dismiss ガード）
```

## 既知の制約 / 今後

- **アプリアイコン（Dock 表示用）**: プロジェクト直下に `icon.png`（正方形・1024px 以上推奨）を置いて
  `build-app.sh` を実行すると、`sips`/`iconutil` で `AppIcon.icns` を自動生成して埋め込みます
  （`icon.png` が無い場合は、置いてある `AppIcon.icns` をそのまま使用）。
- ドラッグ＆ドロップ／フォルダ作成は**ロジック実装・コンパイル・起動までは検証済み**ですが、
  対話操作の挙動は実機での手動確認を推奨（ビルド環境ではフル GUI 自動テスト不可のため）。
- マルチモニターは「カーソルのある画面で開く」方式に対応済み。全画面同時表示
  （全モニターを暗転させる旧挙動）は未対応 — 必要なら追加可能。
- グローバルホットキー（例: F4 で呼び出し）や常駐化は未実装。
  常駐＋ホットキー化やメニューバー常駐が必要なら拡張可能です。
- フル Xcode があれば `.xcodeproj` 化や App Sandbox / 公証（notarization）も可能です。
